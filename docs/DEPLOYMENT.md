# Deployment Guide\n\nThis guide covers deploying your gaming website to production using various platforms.\n\n## Deployment Options\n\n### 1. **Render** (Recommended - Free tier available)\n### 2. **Railway** (Easy setup, pay-as-you-go)\n### 3. **Heroku** (Classic choice, paid)\n### 4. **AWS** (Most scalable)\n### 5. **DigitalOcean** (Affordable VPS)\n\n---\n\n## Option 1: Deploy to Render (Easiest)\n\n### Prerequisites\n- GitHub account\n- Render account (https://render.com)\n\n### Steps\n\n1. **Push to GitHub**\n   ```bash\n   git add .\n   git commit -m \"Ready for deployment\"\n   git push origin main\n   ```\n\n2. **Connect to Render**\n   - Go to https://render.com\n   - Click \"New +\" → \"Web Service\"\n   - Connect your GitHub account\n   - Select `YNGSAVICE/gaming-website`\n   - Choose \"main\" branch\n\n3. **Configure**\n   - **Name:** gaming-website\n   - **Environment:** Docker\n   - **Region:** Choose closest to you\n   - **Plan:** Free (or paid)\n\n4. **Set Environment Variables**\n   - In Render dashboard: Environment → Add Variable\n   - Add these variables:\n     ```\n     FLASK_ENV=production\n     SECRET_KEY=your-random-secret-key\n     JWT_SECRET_KEY=your-jwt-secret\n     DATABASE_URL=postgresql://user:password@host/dbname\n     OPENWEATHER_API_KEY=your-api-key\n     ```\n\n5. **Deploy**\n   - Click \"Deploy\"\n   - Wait for build to complete\n   - Your site will be live at: `https://gaming-website.onrender.com`\n\n---\n\n## Option 2: Deploy to Railway\n\n### Steps\n\n1. **Install Railway CLI**\n   ```bash\n   npm install -g @railway/cli\n   ```\n\n2. **Login to Railway**\n   ```bash\n   railway login\n   ```\n\n3. **Initialize and Deploy**\n   ```bash\n   railway init\n   railway up\n   ```\n\n4. **Set Environment Variables**\n   ```bash\n   railway variables set FLASK_ENV=production\n   railway variables set SECRET_KEY=your-secret\n   railway variables set JWT_SECRET_KEY=your-jwt-secret\n   ```\n\n5. **Get Your URL**\n   ```bash\n   railway open\n   ```\n\n---\n\n## Option 3: Deploy to Heroku\n\n### Steps\n\n1. **Install Heroku CLI**\n   ```bash\n   npm install -g heroku\n   ```\n\n2. **Login**\n   ```bash\n   heroku login\n   ```\n\n3. **Create App**\n   ```bash\n   heroku create gaming-website-unique-name\n   ```\n\n4. **Add PostgreSQL**\n   ```bash\n   heroku addons:create heroku-postgresql:hobby-dev\n   ```\n\n5. **Set Config Variables**\n   ```bash\n   heroku config:set FLASK_ENV=production\n   heroku config:set SECRET_KEY=your-secret-key\n   heroku config:set JWT_SECRET_KEY=your-jwt-secret\n   heroku config:set OPENWEATHER_API_KEY=your-api-key\n   ```\n\n6. **Deploy**\n   ```bash\n   git push heroku main\n   ```\n\n7. **View Logs**\n   ```bash\n   heroku logs --tail\n   ```\n\n---\n\n## Option 4: Deploy to AWS (EC2)\n\n### Steps\n\n1. **Launch EC2 Instance**\n   - Go to AWS Console → EC2\n   - Click \"Launch Instance\"\n   - Choose: Ubuntu 22.04 LTS\n   - Instance Type: t2.micro (free tier)\n   - Security Group: Allow HTTP (80), HTTPS (443), SSH (22)\n\n2. **SSH into Instance**\n   ```bash\n   ssh -i your-key.pem ec2-user@your-instance-ip\n   ```\n\n3. **Install Dependencies**\n   ```bash\n   sudo apt update\n   sudo apt install python3-pip python3-venv postgresql postgresql-contrib nginx git -y\n   ```\n\n4. **Clone Repository**\n   ```bash\n   git clone https://github.com/YNGSAVICE/gaming-website.git\n   cd gaming-website\n   ```\n\n5. **Set up Virtual Environment**\n   ```bash\n   python3 -m venv venv\n   source venv/bin/activate\n   pip install -r requirements.txt\n   ```\n\n6. **Configure Environment**\n   ```bash\n   cp .env.example .env\n   # Edit .env with production values\n   nano .env\n   ```\n\n7. **Setup Nginx Reverse Proxy**\n   ```bash\n   sudo nano /etc/nginx/sites-available/gaming-website\n   ```\n   \n   Add:\n   ```nginx\n   server {\n       listen 80;\n       server_name your-domain.com;\n\n       location / {\n           proxy_pass http://127.0.0.1:5000;\n           proxy_set_header Host $host;\n           proxy_set_header X-Real-IP $remote_addr;\n       }\n   }\n   ```\n\n8. **Enable Site**\n   ```bash\n   sudo ln -s /etc/nginx/sites-available/gaming-website /etc/nginx/sites-enabled/\n   sudo systemctl restart nginx\n   ```\n\n9. **Run with Gunicorn**\n   ```bash\n   pip install gunicorn\n   gunicorn --workers 4 --bind 127.0.0.1:5000 backend.run:app\n   ```\n\n10. **Setup Systemd Service** (auto-restart)\n    ```bash\n    sudo nano /etc/systemd/system/gaming-website.service\n    ```\n    \n    Add:\n    ```ini\n    [Unit]\n    Description=Gaming Website\n    After=network.target\n\n    [Service]\n    User=ubuntu\n    WorkingDirectory=/home/ubuntu/gaming-website\n    ExecStart=/home/ubuntu/gaming-website/venv/bin/gunicorn --workers 4 --bind 127.0.0.1:5000 backend.run:app\n    Restart=always\n\n    [Install]\n    WantedBy=multi-user.target\n    ```\n\n11. **Start Service**\n    ```bash\n    sudo systemctl daemon-reload\n    sudo systemctl start gaming-website\n    sudo systemctl enable gaming-website\n    ```\n\n---\n\n## Option 5: Deploy to DigitalOcean App Platform\n\n### Steps\n\n1. **Go to DigitalOcean**\n   - https://cloud.digitalocean.com/apps\n   - Click \"Create App\"\n\n2. **Connect GitHub**\n   - Authorize DigitalOcean\n   - Select `YNGSAVICE/gaming-website`\n   - Branch: main\n\n3. **Configure**\n   - Name: gaming-website\n   - Source Type: Docker\n   - HTTP Port: 5000\n\n4. **Add Database**\n   - Click \"Add Database\"\n   - Choose PostgreSQL\n\n5. **Environment Variables**\n   - Add all required variables\n\n6. **Deploy**\n   - Review settings\n   - Click \"Deploy\"\n   - Your app will be live in minutes\n\n---\n\n## SSL/HTTPS Setup\n\n### Using Let's Encrypt (Free)\n\n**On AWS/DigitalOcean VPS:**\n```bash\nsudo apt install certbot python3-certbot-nginx -y\nsudo certbot --nginx -d your-domain.com\n```\n\n**Auto-renewal:**\n```bash\nsudo systemctl enable certbot.timer\n```\n\n### On Render/Railway/Heroku\n- Automatic SSL included ✅\n\n---\n\n## Custom Domain Setup\n\n### Step 1: Buy Domain\n- GoDaddy, Namecheap, or Domain.com\n\n### Step 2: Update DNS\n\n**For Render:**\n```\nCNAME record:\nName: @\nValue: cname.onrender.com\n```\n\n**For Railway/Heroku:**\n```\nCNAME record:\nName: @\nValue: [your-app-url]\n```\n\n**For DigitalOcean:**\n```\nCNAME record:\nName: @\nValue: [app-domain]\n```\n\n### Step 3: Configure in Platform Dashboard\n- Add custom domain in your app settings\n- Wait for DNS propagation (up to 48 hours)\n\n---\n\n## Monitoring & Maintenance\n\n### View Logs\n\n**Render:**\n```bash\nVisit Dashboard → Logs\n```\n\n**Heroku:**\n```bash\nheroku logs --tail\n```\n\n**AWS/DigitalOcean:**\n```bash\nsudo journalctl -u gaming-website -f\n```\n\n### Backup Database\n\n**PostgreSQL Backup:**\n```bash\npg_dump $DATABASE_URL > backup.sql\n```\n\n**Restore:**\n```bash\npsql $DATABASE_URL < backup.sql\n```\n\n---\n\n## Performance Tips\n\n1. **Enable Caching**\n   - Add Redis for session caching\n   - Cache static assets with CDN\n\n2. **Use Gunicorn Workers**\n   ```bash\n   gunicorn --workers 4 --worker-class gevent backend.run:app\n   ```\n\n3. **Database Optimization**\n   - Add indexes\n   - Use connection pooling\n\n4. **Static Files**\n   - Use CDN (CloudFlare, AWS CloudFront)\n   - Compress images\n\n5. **Monitor**\n   - Set up alerts for errors\n   - Monitor response times\n   - Track resource usage\n\n---\n\n## Troubleshooting\n\n### \"Application failed to start\"\n- Check logs for error messages\n- Verify DATABASE_URL is correct\n- Ensure all env variables are set\n\n### \"Port already in use\"\n```bash\n# Kill process on port 5000\nlsof -ti:5000 | xargs kill -9\n```\n\n### \"Database connection failed\"\n- Verify PostgreSQL is running\n- Check DATABASE_URL format\n- Ensure firewall allows connections\n\n### \"Static files not loading\"\n- Run: `python -m flask --app backend.run collect-static`\n- Check static folder path\n- Clear browser cache\n\n---\n\n## Support & Resources\n\n- Render Docs: https://render.com/docs\n- Railway Docs: https://docs.railway.app\n- Heroku Docs: https://devcenter.heroku.com\n- AWS Docs: https://docs.aws.amazon.com\n- Flask Deployment: https://flask.palletsprojects.com/deployment\n\n**Recommended:** Start with **Render** (easiest) or **Railway** (most flexible).\n
